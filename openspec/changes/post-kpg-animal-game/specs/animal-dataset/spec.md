## ADDED Requirements

### Requirement: Animal roster of 200 entries
The system SHALL provide a flat JavaScript array `ANIMALS` exported in `animal-game/js/animals.js` containing exactly 200 entries: 100 extant (modern) animals representing the most popularly recognized species alive today, and 100 extinct animals that evolved during the Cenozoic era (after the K-Pg boundary, ~66 Ma ago).

Each entry SHALL conform to the following schema:
```
{
  name: string,       // Common name, title-cased
  wiki: string,       // Wikipedia article key (used to fetch image + facts)
  level: 1–5,         // Difficulty tier (1 = very well known, 5 = obscure)
  length: number,     // Body length in meters (approximate adult average)
  speed: number,      // Top speed in km/h (representative value)
  appeared: number,   // First appearance in millions of years ago (Ma); use 0.0001 for living species with no clear fossil date
  period: string,     // Human-readable time label, e.g. "Pleistocene (2.6–0.01 Ma)" or "Extant"
  diet: string,       // One of: "Carnivore", "Herbivore", "Omnivore", "Piscivore", "Insectivore"
  geo: string,        // Geographic distribution, e.g. "Africa", "Global"
  extinct: boolean    // true for extinct species, false for extant
}
```

#### Scenario: Modern animals coverage
- **WHEN** the `ANIMALS` array is loaded
- **THEN** exactly 100 entries SHALL have `extinct: false`, spanning diverse taxonomic groups (mammals, birds, reptiles, fish, invertebrates) and all major continents

#### Scenario: Extinct Cenozoic animals coverage
- **WHEN** the `ANIMALS` array is loaded
- **THEN** exactly 100 entries SHALL have `extinct: true` and `appeared` values between 0.001 and 66 (Ma), covering major Cenozoic megafauna groups (e.g., proboscideans, terror birds, saber-toothed cats, giant ground sloths, chalicotheres)

#### Scenario: Required fields present
- **WHEN** any entry in `ANIMALS` is inspected
- **THEN** all fields (`name`, `wiki`, `level`, `length`, `speed`, `appeared`, `period`, `diet`, `geo`, `extinct`) SHALL be non-null and of the correct type

### Requirement: Animal image and fact data bundle
The system SHALL provide a `ANIMAL_DATA` object in `animal-game/js/animal-data.js`, auto-generated by `scripts/fetch-animal-data.py`, mapping each animal's `wiki` key to `{ img: string, short: string, full: string }`.

#### Scenario: Data fetch script produces valid output
- **WHEN** `scripts/fetch-animal-data.py` is executed
- **THEN** it SHALL write `animal-game/js/animal-data.js` containing a valid `ANIMAL_DATA` assignment for every entry in `ANIMALS`

#### Scenario: Graceful missing image handling
- **WHEN** an animal's Wikipedia page lacks a usable image
- **THEN** `ANIMAL_DATA[wiki].img` SHALL be an empty string `""` and the game SHALL fall back to text-only question modes for that animal

### Requirement: Difficulty level distribution
The animal roster SHALL distribute difficulty levels so that level-1 animals are universally recognized (lion, elephant, shark) and level-5 animals are known primarily to enthusiasts (e.g., Andrewsarchus, Chalicotherium).

#### Scenario: Level 1 population
- **WHEN** filtering `ANIMALS` by `level === 1`
- **THEN** at least 20 entries SHALL be returned, all with near-universal name recognition

#### Scenario: Level 5 population
- **WHEN** filtering `ANIMALS` by `level === 5`
- **THEN** at least 15 entries SHALL be returned
